## Failure Matrix

| Failure Type         | Injection Point   | Expected Event                          | DropReason       |
|----------------------|-------------------|-----------------------------------------|------------------|
| Malformed packet     | PacketParser      | PacketDropped                           | ParseError       |
| Validation error     | Validator         | PacketDropped                           | ValidationError  |
| Queue overflow       | Queue admission   | PacketDropped                           | QueueOverflow    |
| Simulated packet loss| FailureInjector   | PacketDropped                           | SimulatedLoss    |
| Artificial delay     | FailureInjector   | Non-terminal (delayed lifecycle)        | None             |
| Processing rejection | PacketProcessor   | PacketDropped                           | ProcessingError  |

### Matrix Rules

- Every lifecycle_id must produce exactly one terminal event:
  -> PacketProcessed OR PacketDropped
- Artificial delay MUST NOT create a terminal event.
- No silent drops are permitted.
- lifecycle_id is preserved from injection to terminal event.
- Accounting is based ONLY on lifecycle_id.
- packet.id is payload identity and MUST NOT be used for accounting.
- duplicate_events counts ONLY lifecycle violations.
